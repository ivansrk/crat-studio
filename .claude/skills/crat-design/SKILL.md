# CRAT design: система + процесс

Источник правды: docs/design-brief.md (полный бриф). Здесь — выжимка и обязательный процесс.

## Формула

**Black is the space. Red is the light. Image is the protagonist. Interface is quiet.**
Ночной город, красные вывески, маленький кинотеатр, архив, мастерская, ручная линия. Референсы: MUBI, NOWNESS, Are.na, e-flux, wolfai.be (свет кнопок).

Палитра — только токены lib/design/tokens.css. Красный = свет (glow/underline/линии/прогресс), не заливка. Мята/лаванда — редкие акценты.

## Анти-дефолт (признаки «AI-дизайна» — запрещены)

- Симметричные карточки-коробки 3×N с одинаковыми скруглениями и border на всём
- Синие ссылки, системные фокус-кольца без стилизации, дефолтные чекбоксы
- Градиенты фиолетово-синие «стартаперские», glassmorphism
- Заголовок-по-центру + подзаголовок + две кнопки по центру (шаблонный hero)
- Одинаковый вертикальный ритм всех секций (монотонность); нужен монтаж: чередование плотных и воздушных, полноширинных и колоночных
- emoji в UI, стоковые метафоры (лампочки, ракеты, рукопожатия)
- Текст поверх изображения без scrim/тени → нечитаемость

## Обязательные правила

- Типографика: Cormorant Garamond (display) / Manrope (текст) / JetBrains Mono (label UPPERCASE). База 18–20px, мельче нельзя (аудитория 40+). Строка ≤75ch.
- Тач-таргеты ≥44px. Hover-эффекты — только под `@media (hover: hover)`; `:focus-visible` всегда видим.
- Все строки через lib/i18n/ru.ts — кириллицы в разметке НЕТ.
- CSS-only: Tailwind/framer-motion/сторонние UI-киты запрещены. prefers-reduced-motion глушится глобально — новые анимации обязаны попадать под глушитель.
- Изображения: next/image (lazy, sizes), priority только первому экрану; вес картинок страницы ≤600KB; alt="" для декоративных.
- Белого фона нет нигде. Стоков нет. Картинки — только из public/images (сгенерированные по image-brief) с CSS-эффектами поверх (зерно, glow, scrim).

## Обязательный процесс проверки (без него работа не сдана)

1. Скриншоты Playwright **до и после**: каждая затронутая страница на **360, 768, 1024, 1440** — просмотреть глазами каждый (не «прогнал и ок»).
2. Замеры при любых фиксированных высотах/overflow: контент ≤ контейнера на всех ширинах (getBoundingClientRect-таблица в отчёт).
3. `document.body.scrollWidth <= innerWidth` на 360 (нет горизонтального скролла).
4. Проверить состояния: hover, :focus-visible (клавиатура), checked, disabled, пустые данные.
5. Гейты: typecheck / lint / test / build.
6. В отчёте не заявлять «проверено», если не смотрел скриншот, — финальные ревью делают hit-тесты и ловят ложные заявления.

## История пойманных дефектов (не повторять)

- aspect-ratio + overflow:hidden обрезал контент карточек (нужен min-height, рост под контент)
- flex-basis стал высотой при flex-direction:column (баннер в полэкрана)
- Отрицательный margin-block создал перекрытие кликабельных зон nav при переносе строк
- grid gap на контейнере прозы раздул межабзацные интервалы MDX
- Центрированный логотип «повисал» на мобильном при многострочной подписи
