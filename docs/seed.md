# scripts/seed.ts — описание

Назначение: за один запуск (`npm run seed`) привести базу к состоянию, в котором Иван может проверить текущую фазу за 10 минут, не собирая данные руками. Используется локально и на Render (после деплоя фазы).

Требования к скрипту:
- **Идемпотентен**: повторный запуск не создаёт дублей (везде upsert по естественным ключам: email, (userId, lessonId) и т.д.).
- Не трогает реальных пользователей: работает только с адресами `*@seed.crat.example`.
- Расширяется по фазам — каждая фаза добавляет свой блок, старые блоки не ломаются.

## Что создаёт (по фазам)

**Ф0:**
- Админ-пользователей из `ADMIN_EMAILS` (это же делает приложение при старте — seed проверяет/чинит).

**Ф1:**
- Заявку `zayavka@seed.crat.example` (статус NEW, оба согласия).
- Повторную заявку `povtor@seed.crat.example` (RESUBMITTED, submitCount=2, без согласия на рассылку — для проверки CSV-фильтра).
- Студента `student@seed.crat.example`: User + Enrollment + согласия (ENROLLED-заявка).

**Ф2:**
- Студенту прогресс: уроки 1.1–1.2 пройдены (квиз-попытки: одна проваленная, одна зачтённая; практика отмечена), урок 1.3 — квиз зачтён, практика нет; заполненная личная миссия; одна брошенная попытка квиза (finishedAt=null).

**Ф3:**
- Студента `diplomant@seed.crat.example` с 12/12 пройденными уроками и Submission SUBMITTED — одна кнопка approve отделяет от проверки выдачи сертификата.
- Submission NEEDS_CHANGES с комментарием у `student@seed.crat.example`.

**Ф4:**
- Бэкдейт: у `student@seed.crat.example` completedAt урока 1.1 = −8 дней, deferred dueAt = −1 день → блок «3 вопроса на 2 минуты» виден сразу.
- Записи trainer_usage на грани дневного лимита (19/20) для проверки мягкого отказа.

**Ф5:** новых данных не требует.

**Ф7а:**
- Известный пароль (`SEED_PASSWORD`, см. ниже) всем seed-юзерам: `student@`, `diplomant@`, всем `ADMIN_EMAILS`.

## Вход seed-юзерами (кабинет и админка)

Ф7а (D-031): вход только по email + паролю, magic-link снят. Seed проставляет ВСЕМ seed-юзерам
(студенту `student@seed.crat.example`, дипломанту `diplomant@seed.crat.example` и всем адресам
из `ADMIN_EMAILS`) один и тот же известный пароль — идемпотентно: повторный `npm run seed`
восстанавливает его, даже если пароль поменяли руками при ручном тестировании.

- **Пароль:** `seed-pass-2026` (константа `SEED_PASSWORD` в `scripts/seed.ts`).
- Seed печатает в stdout каждую пару `email / пароль` — копировать оттуда, не хранить отдельно.
- Логин: `/login` → email + пароль → редирект в `/app` (студент/дипломант) или `/admin` (админ-email).
- Работает и локально, и на стенде Render (после `npm run seed` там же) — пароль один и тот же,
  т.к. это не боевые учётки, а вымышленные `*@seed.crat.example` + реальные ADMIN_EMAILS с
  тестовым паролем (сменить в кабинете, если стенд смотрят посторонние).

Seed также печатает одну демонстрационную reset-ссылку (`[seed reset-demo]`, для `student@`) —
живой пример флоу «Забыли пароль» (F12/AUTH-16/17) без настоящего Resend; на вход основным
паролем эта ссылка не влияет.

**Ф7б:**
- Инвайт-ссылка на курс `ai-basics` (`sourceLabel: 'seed-invite'`, без лимита и срока) — для
  проверки формы регистрации по `/invite/{token}` и авто-выдачи доступа (INV-01…06).
- Заявка `optin@seed.crat.example` в статусе `PENDING_OPT_IN` (с телефоном и `wantsNewsletter=true`)
  и живым OPT_IN-токеном — для ручной проверки полного цикла double opt-in: форма уже «отправлена»,
  осталось только подтвердить.
- Подписанный клиент `confirmed@seed.crat.example`: User с действующими Consent DATA_PROCESSING
  и NEWSLETTER (granted) — виден в `/admin/clients` с «Да» в колонке «Подписан» (CRM-01/02).
- Заявка на консультацию от `student@seed.crat.example` в статусе `NEW`, источник `cabinet` —
  видна в `/admin/consultations` (CONS-01…04).

Seed печатает в stdout две ссылки для ручной проверки:
- `[seed] инвайт: {url}` — открыть, заполнить форму регистрации, отправить.
- `[seed] confirm: {url}` — ссылка подтверждения для заявки `optin@`; письмо DOUBLE_OPT_IN реально
  НЕ уходит (Resend не настроен локально) — вместо похода в почту используем эту ссылку из stdout,
  чтобы пройти double opt-in целиком: заявка → PENDING_OPT_IN → переход по `[seed] confirm` →
  CONFIRMED/ENROLLED (в зависимости от того, был ли инвайт).

**Как проверить (10 минут):** открыть `[seed] инвайт`-ссылку → заполнить и отправить форму
регистрации → зайти в `/admin` и убедиться, что новая заявка видна; открыть `[seed] confirm`-ссылку
для `optin@` и убедиться, что заявка перешла в `CONFIRMED`/`ENROLLED`; открыть `/admin/clients` —
`confirmed@seed.crat.example` виден с «Да» в «Подписан»; открыть `/admin/consultations` —
заявка от `student@` в статусе `NEW`.

**Ф7в:** новых seed-данных не требует. `Enrollment.courseSlug` по умолчанию `'ai-basics'` (Prisma
`@default`) — все существующие блоки (F1–F7б) продолжают писать прогресс/заявки/сертификаты именно
на этот курс без правок. Второй, демонстрационный курс `content/demo-course` (`published: false`)
сознательно НЕ получает enrollment ни от одного seed-юзера — до публикации он должен быть виден в
каталоге хаба `/app` как «Скоро», а не как «Мой курс» с прогрессом. Печатаемые в stdout ссылки
(`reset-demo`, `инвайт`, `confirm`) курсо-независимы (`/reset/…`, `/invite/…`, `/invite-confirm/…`) —
путей вида `/app/lessons/…` в них не было и после мультикурса не появилось, править нечего.

**Как проверить мультикурс (Ф7в, 10 минут):**

- Войти `student@seed.crat.example` / `seed-pass-2026` → `/app` показывает хаб: карточка
  `ai-basics` в «Мои курсы» с ненулевым прогрессом (2/12 по данным Ф2), `demo-course` — в каталоге
  остальных курсов с пометкой «Скоро» (не published).
- Открыть карточку `ai-basics` → `/app/ai-basics` — горизонт-прогресс по модулям, урок открывается
  и проходится как раньше.
- Старый URL `/app/lessons/1.1` (и `/app/certificate`, `/app/project`) отвечает 308 на
  `/app/ai-basics/…` — закладки и письма, выданные до мультикурса, не ломаются.
- `/health` зелёный по обоим курсам реестра: `ai-basics` (12 уроков) и `demo-course`.
- Футер → `/privacy`, `/terms`, `/cookies` открываются (200, заглушки с разделами); куки-баннер
  показывается один раз и не возвращается после закрытия.
- Главная `/` — 200, изображения (hero/направления/студия) грузятся.
