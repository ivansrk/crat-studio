# Контракт контента: course-factory → платформа

**Версия контракта: v2 (2026-07-08), согласован с course-factory** (запись в DECISIONS.md и CLAUDE.md фабрики от 2026-07-08). Изменения v2: урок — `lesson.mdx` вместо `text.md` (белый список компонентов, раздел 4), каталог `assets/` в уроке, раздел статей `content/articles/`.

Контент производится в репозитории course-factory и доставляется файлами в `content/{course-slug}/` этого репозитория (копирование/PR из output фабрики). **Обновление урока = git push** → деплой Render → валидация при старте. Контент в базе не хранится (D-002).

Исходники материалов (картинки, PDF) команда ведёт в Google Drive (`edu@skld.me`); в git они попадают скриптом синхронизации на стороне course-factory (D-023). Платформа раздаёт файлы сама — прямых ссылок на Drive в контенте нет.

Изменения этого контракта согласовываются с course-factory — в одностороннем порядке не менять (правило 7 CLAUDE.md).

## 1. Структура каталога

```
content/
  ai-basics/
    course.yaml                 # паспорт курса — источник правды о составе
    module-1/
      lesson-1.1/
        meta.yaml               # обязателен
        lesson.mdx              # обязателен (v1: text.md)
        quiz.yaml               # обязателен
        practice.md             # обязателен
        cheatsheet.pdf          # опционален (LES-05)
        assets/                 # опционален: картинки, PDF, на которые ссылается lesson.mdx
      lesson-1.2/
      ...
    module-2/ … module-4/
  articles/                     # статьи для /articles (SEO/GEO), см. раздел 8
    kak-vybrat-neyroset/
      meta.yaml
      article.mdx
      assets/
```

Идентификатор урока — `{модуль}.{номер}` («1.1» … «4.3»). Он же — ключ `lessonId` в базе (LessonProgress, QuizResult и т.д.), поэтому **id уроков после запуска менять нельзя** — это разрыв прогресса студентов.

## 2. course.yaml

```yaml
slug: ai-basics
title: "Искусственный интеллект в профессиональной и личной деятельности"
modules:
  - id: 1
    title: "Название модуля 1"
    lessons:
      - id: "1.1"
        title: "Название урока"
      - id: "1.2"
        title: "…"
  # … всего 4 модуля, 12 уроков
```

Правила:
- `slug` совпадает с именем каталога.
- Список уроков в course.yaml — **единственный источник** состава курса: по нему строится кабинет и считается условие сертификата «все 12» (CERT-01).
- Урок, лежащий на диске, но не указанный в course.yaml, игнорируется (с warning в лог).

## 3. meta.yaml (урок)

```yaml
id: "1.1"                # обязателен, = id в course.yaml и имени каталога
title: "Название урока"  # обязателен
video_id: "abc123"       # Kinescope id; пустая строка/отсутствие = плейсхолдер (LES-03)
duration_min: 12         # опционально, для отображения
cheatsheet: true         # опционально; true → рядом должен лежать cheatsheet.pdf
mission_prompt: false    # true только в уроке 1.1: показать поле «личная миссия» (LES-14)
```

## 4. lesson.mdx / practice.md

- База — GitHub-flavored markdown, язык — русский; поверх — **только** компоненты из белого списка ниже (LES-15).
- Заголовки начинаются с `##` (`#` зарезервирован под title из meta.yaml).
- Изображения — относительные пути внутри каталога урока (`![схема](assets/scheme.png)` или `<Figure src="assets/scheme.png" …/>`). Файлы лежат **плоско** прямо в `assets/` — подпапки (`assets/img/…`) не поддерживаются и дадут ошибку валидации.
- **Запрещено:** произвольный JSX/JS (import, export, выражения `{…}` — в том числе в атрибутах и spread `{...props}`), любые HTML/JSX-теги вне белого списка (включая обычные `<b>`, `<div>`, а не только script/iframe/style). Нарушение = ошибка валидации. Форматирование — средствами markdown.
- practice.md — обычный markdown (без компонентов); текст задания; чекбокс «Сделал» добавляет платформа, в файле его нет.

### Белый список компонентов v2

| Компонент | Пропсы | Что делает |
| --- | --- | --- |
| `<Figure src caption? />` | src — путь в assets/, caption — подпись | Иллюстрация с подписью, крупно |
| `<Gallery>…</Gallery>` | children: несколько `<Figure>` (только так; props у Gallery нет) | Галерея из 2–6 изображений |
| `<Callout type>текст</Callout>` | type: `idea` \| `warning` \| `example` | Врезка в стиле CRAT |
| `<Video kinescope />` | kinescope — id ролика | Kinescope-плеер в любом месте урока |
| `<Download file>подпись</Download>` | file — путь в assets/ или cheatsheet.pdf | Кнопка скачивания PDF/файла |
| `<Trainer id mode? />` | id: `t1` \| `t2` \| `t3`; mode: `inline` (по умолч.) \| `link` | Тренажёр в уроке (TRN-07) |
| `<Animation id />` | id — из библиотеки анимаций платформы (`lib/design/animations`) | Анимация/движущийся элемент; фабрика не пишет свои — только ссылается |
| `<Divider />` | — | Декоративный разделитель секций |

Список расширяется только парой «запись в этом файле + реализация в платформе»; компонент, которого нет в списке, — ошибка валидации, а не «молча не отрендерился».

## 6. Валидация (lib/content, при старте приложения — CNT-02)

Ошибка (урок помечается недоступным, лог + баннер в админке, приложение работает — CNT-03):
- course.yaml отсутствует / не парсится / slug ≠ каталогу / дубли id уроков;
- у урока из course.yaml нет каталога или любого из обязательных файлов (meta.yaml, lesson.mdx, quiz.yaml, practice.md);
- yaml не парсится; `meta.id` ≠ id каталога/course.yaml;
- lesson.mdx не компилируется; содержит компонент вне белого списка; import/export/JS-выражения; `<Trainer id>` с неизвестным id; `<Animation id>` без записи в библиотеке анимаций;
- `<Figure>`, `<Download>`, `![…]` ссылаются на файл, которого нет в assets/ (или cheatsheet.pdf отсутствует);
- в quiz.yaml не ровно 3 вопроса; в вопросе <2 или >4 вариантов; `correct` вне диапазона; пустые `question`/`explanation`;
- `cheatsheet: true`, а cheatsheet.pdf отсутствует.

Warning (лог, не блокирует урок):
- пустой `video_id` (штатная ситуация — LES-03);
- каталог урока без записи в course.yaml;
- отсутствие `deferred` (штатно — D-012).

Поведение при ошибках:
- Валидация выполняется целиком, собираются ВСЕ ошибки (не первая).
- Битый урок при существующем прогрессе: прогресс не трогается, страница урока — «Урок временно недоступен, мы уже чиним».
- Приложение никогда не падает из-за контента.

## 8. Статьи (`content/articles/`, раздел ART требований)

Статья = каталог `content/articles/{slug}/`:

```yaml
# meta.yaml
slug: kak-vybrat-neyroset      # = имени каталога; часть URL /articles/{slug}
title: "Как выбрать нейросеть"
description: "1–2 предложения для сниппета поиска и OG"  # обязателен (SEO)
date: 2026-08-01               # дата публикации
draft: false                   # true → не публикуется и не попадает в sitemap
cover: kak-vybrat-neyroset.webp  # опционально (Ф7в, аддитивно) — имя файла в public/images/
```

- `article.mdx` — тот же формат и белый список, что lesson.mdx, но `<Trainer>` в статьях запрещён (тренажёры — только студентам, TRN-06).
- `cover` — необязательное поле (Ф7в, D-038): имя файла обложки в `public/images/`, 16:9. Отсутствует → карточка/шапка статьи без картинки, как раньше. Файл не найден на диске — warning-issue (лог/баннер в админке), статья публикуется как обычно, без cover. Поле аддитивно к контракту — согласовано с course-factory при встройке.
- Валидация — те же правила, что для уроков: битая статья не публикуется (лог + баннер в админке), сайт работает.
