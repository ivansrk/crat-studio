# Фазы разработки

Правила:
- Фаза закрывается только когда: критерий готовности выполнен, seed обновлён (docs/seed.md), Иван прошёл smoke-чеклист (~10 минут).
- «Требуется от Ивана» — внешние зависимости; их лучше запускать заранее, они не блокируют начало фазы, но блокируют её закрытие.

## Ф0 — Скелет + деплой (1 день)

**Объём:** репозиторий Next.js (App Router, TS) · Prisma + начальная миграция всей схемы (docs/data-model.md) · Render: web service + managed Postgres, автобэкапы, миграции в deploy-команде · `.env.example`, gitleaks + pre-commit · GitHub Actions CI (typecheck, lint, gitleaks) · дизайн-токены и шрифты как CSS-переменные (UX-03, UX-04) · словарь `lib/i18n/ru.ts` · модуль `lib/content` с валидацией контракта v2: **MDX-движок с белым списком компонентов** + стартовая библиотека анимаций `lib/design/animations` (CNT-02, CNT-03, LES-04, LES-15, LES-16) · заглушки 12 уроков `content/ai-basics/` в MDX, каждый компонент белого списка задействован хотя бы раз (CNT-04) · Plausible · Sentry · страница `/health` (аптайм + статус валидации контента) · seed v0.

**Критерий готовности:** прод-URL на Render открывается; `/health` зелёный; список модулей/уроков рендерится из content/; битый yaml в тестовом уроке даёт заглушку, а не падение.

**Smoke-чеклист Ивана:**
1. Открыть прод-URL — страница загружается, шрифты/цвета CRAT (тёмный фон, не белый).
2. Открыть `/health` — ok, контент валиден.
3. Открыть любой урок-заглушку — текст читается, кегль крупный.

**Требуется от Ивана:** аккаунт Render (или доступ), выбор домена платформы (APP_URL). ⚠️ Уже сейчас запустить настройку Resend (см. Ф1).

## Ф1 — Регистрация + magic link + админка

**⚠️ ТРЕБУЕТСЯ ОТ ИВАНА (внешний блокер):** домен + DKIM настроить в Resend до закрытия Ф1 (MAIL-02). Без этого письма не уходят — вся фаза проверяется только на тестовом inbox.

**Объём:** главная CRAT studio и лендинг курса с программой из course.yaml (SITE-01…06; тексты-черновики, финальные — от Ивана, визуальная полировка — Ф5) · SEO-база публичных страниц: SSR, метаданные/OG, sitemap.xml, robots.txt, canonical (SEO-01…03, SEO-07) · публичная регистрация — форма на лендинге (REG-01…09, rate limit) · согласия в consents · magic link вход (AUTH-01…11, F3) · сессии-cookie · синк админов из ADMIN_EMAILS · админка: заявки, студенты, «выдать доступ» (ADM-01…04), переотправка писем (ADM-08), CSV-экспорт (ADM-09), GDPR-удаление (ADM-10, ADM-11; отзыв сертификата активируется в Ф3) · email_log + ретраи (MAIL-03…05) · письма MAGIC_LINK и ACCESS_GRANTED · `/unsubscribe/{токен}` (MAIL-06).

**Критерий готовности:** полный цикл F1→F2→F3 работает на проде: заявка → выдача доступа → письмо → вход по ссылке → кабинет-каркас.

**Smoke-чеклист Ивана:**
0. Открыть главную → перейти на лендинг курса → программа из 4 модулей на месте.
1. На лендинге оставить заявку (оба чекбокса) → «Заявка принята».
2. Отправить ту же форму повторно → в админке одна заявка с пометкой повторной.
3. В админке «выдать доступ» → письмо пришло → вход по ссылке.
4. Кликнуть ссылку второй раз → «ссылка использована».
5. Экспорт CSV → только согласившиеся на рассылку.
6. GDPR-удалить тестового студента → студент исчез, вход не работает.

**Требуется от Ивана:** Resend (домен+DKIM), список ADMIN_EMAILS, текст согласия на обработку ПД, тексты главной и лендинга (до финальных текстов работают черновики).

## Ф2 — Уроки + квизы + прогресс

**Объём:** страница урока целиком (LES-01…16): Kinescope embed/плейсхолдер, MDX-контент со всеми компонентами белого списка, шпаргалка, кнопка «Завершить урок» → квиз отдельным шагом, мгновенные пояснения и пересдачи, практика-чекбокс, «пройден» = квиз+практика · кабинет: прогресс, список уроков, личная миссия (CAB-01…03; визуализация — простая, арт в Ф5) · прогресс студентов в админке (ADM-05) · создание deferred_quiz_state при прохождении (LES-13; показ блока — Ф4).

**Критерий готовности:** тестовый студент проходит урок от видео до «пройден»; прогресс виден студенту и админу; квиз, брошенный на середине, ничего не ломает.

**Smoke-чеклист Ивана:**
1. Открыть урок 1.1 → заполнить миссию → она видна в кабинете, редактируется.
2. Нажать «Завершить урок» → квиз открылся; провалить (0–1/3) → пересдать на ≥2/3 → квиз зачтён.
3. Закрыть квиз на 2-м вопросе, вернуться → новая попытка с 1-го вопроса.
4. Отметить практику → урок «пройден», фигурка сдвинулась.
5. Урок без video_id → плейсхолдер, всё остальное работает.
6. В админке виден прогресс тестового студента.

**Требуется от Ивана:** тестовый video_id Kinescope; первые реальные уроки из course-factory (можно позже — работает на заглушках).

## Ф3 — Мини-проект + сертификат + письма

**Объём:** форма мини-проекта, статусы, пересдачи (PROJ-01…06) · проверка в админке с optimistic concurrency (ADM-06, ADM-07) · выдача сертификата: условие, нумерация, PDF Playwright, письмо, кабинет (CERT-01…05) · публичная проверка `/cert/{номер}` (CERT-06) · отзыв при GDPR (CERT-07 — дополняет Ф1) · письмо CERTIFICATE.

**Критерий готовности:** полный путь до сертификата: 12/12 + approved → PDF в письме и кабинете → `/cert/{номер}` подтверждает; GDPR-удаление показывает «отозван».

**Smoke-чеклист Ивана:**
1. Сдать мини-проект → «на проверке».
2. Поставить needs_changes с комментарием → студент видит комментарий, пересдаёт.
3. Approve при 11/12 уроках → сертификата нет; пройти 12-й → сертификат выдан.
4. Письмо с PDF пришло; PDF скачивается из кабинета; номер формата CRAT-2026-NNNN.
5. `/cert/{номер}` — действителен; после GDPR-удаления — «отозван по запросу владельца».

**Требуется от Ивана:** утверждение текста писем и плейсхолдер-макета сертификата.

## Ф4 — Отложенные квизы + тренажёры (T1)

**Объём:** блок «3 вопроса на 2 минуты» при входе, без cron (CAB-04…06, D-005) · каркас `/trainers/{id}` с заглушками T1/T2/T3 (TRN-01) · каталог тренажёров в кабинете `/app/trainers`, только студентам, без геймификации (TRN-06) · компонент `<Trainer>` в уроках с общими лимитами (TRN-07) · серверный прокси Anthropic для T1: max_tokens 1000, 20/день, 3/мин, мягкие отказы (TRN-02…05).

**Критерий готовности:** студент с уроком, пройденным ≥7 дней назад (в seed — бэкдейт), видит и сдаёт блок повторения; T1-прокси отвечает и режет лимиты.

**Smoke-чеклист Ивана:**
1. Войти seed-студентом → блок «3 вопроса на 2 минуты» → ответить → блок исчез, статусы уроков не изменились.
2. Открыть /trainers/t1 → отправить запрос → ответ модели.
3. Отправить 4 запроса за минуту → мягкое сообщение о лимите.

**Требуется от Ивана:** ANTHROPIC_API_KEY; спеки тренажёров из course-factory (для полноценного T1; каркас работает без них).

## Ф6 — Запуск-пакет: статьи + полный SEO/GEO

**Объём:** раздел `/articles` на MDX-движке (ART-01…04) · schema.org JSON-LD: Organization, Course, Article, FAQPage (SEO-04) · `/llms.txt` из контента (SEO-05) · OG-картинки в стиле CRAT · аудит Core Web Vitals mobile (SEO-06) · подтверждение домена и отправка sitemap в Google Search Console и Яндекс.Вебмастер (SEO-08) · UptimeRobot на `/health` · Drive-синк материалов работает на стороне course-factory (DRV-01, D-023).

**Критерий готовности:** 5–10 статей опубликованы; sitemap принят обоими вебмастерами; Lighthouse mobile главной и лендинга — «good» по CWV; llms.txt отдаётся и актуален.

**Smoke-чеклист Ивана:**
1. Открыть `/articles` → статьи читаются, выглядят в стиле CRAT.
2. Вставить URL лендинга в валидатор Rich Results — Course распознан.
3. Открыть `/llms.txt` и `/sitemap.xml` — данные актуальны.
4. Search Console и Вебмастер видят сайт, ошибок индексации нет.

**Требуется от Ивана:** приёмка 5–10 статей от course-factory (пишет фабрика — решение 2026-07-08); доступ к Google Search Console / Яндекс.Вебмастеру; папки Drive для синка согласованы с course-factory.

## Ф7 — Оплата Stripe

**Объём:** Stripe Checkout на лендинге (PAY-01) · webhook с проверкой подписи, идемпотентность, авто-enrollment `source=stripe` + письмо со входом (PAY-02) · ручная выдача остаётся (PAY-03) · страница «платежи» в админке со сверкой по Stripe API (PAY-04) · env `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET` (PAY-05) · таблица платежей — спецификация в docs/data-model.md перед началом фазы.

**Критерий готовности:** тестовая оплата (Stripe test mode) от клика на лендинге до письма со входом без участия админа; повторный webhook не создаёт дублей.

**Smoke-чеклист Ивана:**
1. Оплатить тестовой картой → письмо пришло → вход работает, курс доступен.
2. В админке платёж виден, статус совпадает со Stripe.
3. Выдать доступ вручную другому email — работает как раньше.

**Требуется от Ивана:** ключи Stripe (тот же аккаунт, что у Тильды — не конфликтует), цена курса, решение по валюте.
