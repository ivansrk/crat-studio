# Фазы разработки

Правила:
- Фаза закрывается только когда: критерий готовности выполнен, seed обновлён (docs/seed.md), Иван прошёл smoke-чеклист (~10 минут).

**Критерий готовности:** прод-URL на Render открывается; `/health` зелёный; список модулей/уроков рендерится из content/; битый yaml в тестовом уроке даёт заглушку, а не падение.

**Smoke-чеклист Ивана:**

2. Открыть `/health` — ok, контент валиден.

**Требуется от Ивана:** аккаунт Render (или доступ), выбор домена платформы (APP_URL). ⚠️ Уже сейчас запустить настройку Resend (см. Ф1).

## Ф2 — Уроки + квизы + прогресс

**Объём:** страница урока целиком (LES-01…16): Kinescope embed/плейсхолдер, MDX-контент со всеми компонентами белого списка, шпаргалка, кнопка «Завершить урок» → квиз отдельным шагом, мгновенные пояснения и пересдачи, практика-чекбокс, «пройден» = квиз+практика · кабинет: прогресс, список уроков, личная миссия (CAB-01…03; визуализация — простая, арт в Ф5) · прогресс студентов в админке (ADM-05) · создание deferred_quiz_state при прохождении (LES-13; показ блока — Ф4).

**Критерий готовности:** тестовый студент проходит урок от видео до «пройден»; прогресс виден студенту и админу; квиз, брошенный на середине, ничего не ломает.

**Smoke-чеклист Ивана:**
1. Открыть урок 1.1 → заполнить миссию → она видна в кабинете, редактируется.
2. Нажать «Завершить урок» → квиз открылся; провалить (0–1/3) → пересдать на ≥2/3 → квиз зачтён.
3. Закрыть квиз на 2-м вопросе, вернуться → новая попытка с 1-го вопроса.
4. Отметить практику → урок «пройден», фигурка сдвинулась.
5. Урок без video_id → плейсхолдер, всё остальное работает.
6. В админке виден прогресс тестового студента.

**Требуется от Ивана:** тестовый video_id Kinescope; первые реальные уроки из course-factory (можно позже — работает на заглушках).

## Ф3 — Мини-проект + сертификат + письма

**Объём:** форма мини-проекта, статусы, пересдачи (PROJ-01…06) · проверка в админке с optimistic concurrency (ADM-06, ADM-07) · выдача сертификата: условие, нумерация, PDF Playwright, письмо, кабинет (CERT-01…05) · публичная проверка `/cert/{номер}` (CERT-06) · отзыв при GDPR (CERT-07 — дополняет Ф1) · письмо CERTIFICATE.

**Критерий готовности:** полный путь до сертификата: 12/12 + approved → PDF в письме и кабинете → `/cert/{номер}` подтверждает; GDPR-удаление показывает «отозван».

**Smoke-чеклист Ивана:**
1. Сдать мини-проект → «на проверке».
2. Поставить needs_changes с комментарием → студент видит комментарий, пересдаёт.
3. Approve при 11/12 уроках → сертификата нет; пройти 12-й → сертификат выдан.
4. Письмо с PDF пришло; PDF скачивается из кабинета; номер формата CRAT-2026-NNNN.
5. `/cert/{номер}` — действителен; после GDPR-удаления — «отозван по запросу владельца».

**Требуется от Ивана:** утверждение текста писем и плейсхолдер-макета сертификата.

### Ф7а — Вход по паролю (заменяет magic link)

**Объём:** email+пароль вход (AUTH-12/13, F10) · bcryptjs-хэш cost≥12 (AUTH-14, D-032) · генерация человекочитаемого пароля ≥12 при создании учётки, показ один раз + письмо `WELCOME` (AUTH-15, F11) · «Забыли пароль»/reset на переиспользованной token-механике (MagicLink→PasswordResetToken, TTL 60м; AUTH-16/17, F12) · смена пароля в кабинете `/app/account` (AUTH-18, F13) · миграция существующих юзеров: `passwordHash=null` + разовая рассылка set-password + fallback на `/login` (AUTH-19, D-034, F12 п.3) · rate-limit перебора пароля (AUTH-20) · правка CLAUDE.md (принцип «вход без паролей» → «вход по паролю») · обновление seed (пароли seed-юзерам). **Отменяет:** magic-link-вход (AUTH-01/02, F3) — код входа по ссылке снимается, письмо MAGIC_LINK как способ входа не отправляется.

**Критерий готовности:** новый и существующий пользователь входят по паролю; reset-цикл работает; magic-link-входа больше нет; на проде существующие юзеры получили путь установки пароля.

**Smoke-чеклист Ивана:**
1. Выдать доступ тестовому email → письмо с паролем → вход по email+паролю → кабинет.
2. «Забыли пароль» → письмо → задать новый → вход новым паролем.
3. Сменить пароль в кабинете → старый не работает, новый работает.
4. Существующий (до Ф7) юзер: «первый вход/забыли пароль» → задаёт пароль → входит.
5. 11 неверных паролей подряд → мягкое «слишком много попыток».

**Требуется от Ивана:** утвердить тексты писем WELCOME/PASSWORD_RESET; подтвердить формат генерируемого пароля; согласие на разовую рассылку set-password существующим юзерам (когда на проде появятся реальные).

### Ф7б — Приток клиентов: инвайты, double opt-in, CRM, консультации

**Объём:** инвайт-ссылки в админке (INV-01…06, `/admin/invites`) · расширенная форма регистрации: телефон обяз., whatsapp, double opt-in (REG-10…16, F14/F15) · авто-выдача по инвайту / ручная по публичной заявке (D-035) · письмо `DOUBLE_OPT_IN` · клиентская база `/admin/clients` с поиском и карточкой-редактированием (CRM-01…03, F16) · Resend Broadcasts: живой синк подписчиков в Audience при подтверждении/отписке/GDPR (CRM-04…07, F17; env `RESEND_AUDIENCE_ID`) · консультации: форма (кабинет+публично), `ConsultationRequest`, письмо `CONSULTATION` админам, `/admin/consultations` со статусами (CONS-01…06, F18) · обновление seed (инвайт, неподтверждённая заявка, клиент, консультация).

**Критерий готовности:** по инвайту гость регистрируется → подтверждает email → получает доступ+пароль автоматически; подписчик появляется в Resend Audience; консультация из кабинета доходит письмом и видна админу.

**Smoke-чеклист Ивана:**
1. Создать инвайт в админке → открыть ссылку → заполнить форму (телефон обяз.) → «проверьте почту».
2. Перейти по письму подтверждения → доступ выдан автоматически, пароль показан, курс открыт.
3. Отозвать инвайт → ссылка «недействительна».
4. Публичная заявка с /ai-basics → подтверждение → «доступ выдаст администратор» (не авто).
5. `/admin/clients` → найти клиента, отредактировать телефон; проверить, что подписчик есть в Resend Audience.
6. Оставить консультацию из кабинета → письмо админам → статус в `/admin/consultations`.

**Требуется от Ивана:** `RESEND_AUDIENCE_ID` (создать Audience в Resend); утвердить тексты письма подтверждения и оффера консультаций; решение по D-035 (публичная заявка — ручная выдача, инвайт — авто).
