# CRAT studio — сайт студии + платформа онлайн-курсов

Один сайт (D-017): главная CRAT studio (`/`) · лендинг курса с формой заявки (`/ai-basics`) · статьи для SEO/GEO (`/articles`, Ф6) · ЛМС — кабинет студента и админка (`/app`, `/admin`). Первый курс: «Искусственный интеллект в профессиональной и личной деятельности» (slug: `ai-basics`) — 4 модуля, 12 уроков, финальный мини-проект, именной сертификат.

Аудитория — взрослые без технической подготовки. Следствия для UX (обязательны):
- крупный кегль: base 18–20px, мелкий текст запрещён
- одна страница = одно действие
- вход по email+паролю; пароль генерирует сервер и показывает один раз, восстановление — по email-ссылке (D-031)

## Статус проекта

**Спеки → явное утверждение Ивана («спеки утверждаю») → код. Никакого кода до утверждения.**
Текущая фаза — см. docs/phases.md.

## Стек (зафиксирован, не менять без записи в DECISIONS.md)

- Next.js (App Router) + TypeScript
- PostgreSQL + Prisma; миграции выполняются в deploy-команде Render (`prisma migrate deploy`)
- Хостинг: Render — web service + managed Postgres (автобэкапы включить)
- Письма: Resend · PDF: Playwright + Chromium (HTML-шаблон → PDF, системные шрифтовые fallback)
- Аналитика: Plausible (cookieless)
- Контент — файлы в `content/`, НЕ в базе; обновление урока = git push. Урок — MDX с белым списком компонентов (контракт v2, D-018)
- Без Stripe в MVP (Ф0–Ф5); Stripe Checkout — Ф7 (D-024); в `Enrollment.source` заложено поле под источники
- Материалы команда ведёт в Google Drive (`edu@skld.me`); синк в git — на стороне course-factory (D-023); студентам файлы раздаёт только наш сервер

## Структура

```
app/              # роуты Next.js App Router (тонкие: разбор запроса + вызов lib/)
lib/              # доменные модули (глубокие, по Ousterhout)
  auth/           # email+пароль, сессии, роли
  content/        # загрузка и валидация content/, единственная точка чтения контента
  progress/       # прогресс, квизы, отложенные вопросы, статус «пройден»
  cert/           # номера, генерация PDF, проверка, отзыв
  email/          # Resend, email_log, ретраи, переотправка
  i18n/           # словарь строк (ru.ts); строк в разметке НЕТ
  design/         # токены (см. docs/requirements.md §11)
prisma/           # schema.prisma + миграции
content/ai-basics/ # контент курса (контракт v2: docs/content-format.md)
content/articles/  # статьи /articles (Ф6, тот же MDX-движок)
scripts/          # seed.ts (docs/seed.md)
docs/             # спецификации — источник правды
public/
```

## Правила

1. **Спеки до кода.** Требования — в EARS в docs/requirements.md. Меняется поведение → сначала правка требования.
2. **DECISIONS.md.** Каждое архитектурное/продуктовое решение — запись: что, почему, отвергнутые альтернативы. Спорное решение без согласования Ивана не принимать молча — пометка [РЕШЕНИЕ АВТОРА] и вопрос.
3. **Секреты — никогда в репо.** `.env.example` содержит только имена переменных. Pre-commit c gitleaks обязателен. `ANTHROPIC_API_KEY` используется только на сервере.
4. **i18n-ready.** Все строки интерфейса — в `lib/i18n/`, MVP только русский.
5. **Часовой пояс.** В базе — UTC. Отображение и бизнес-правила («7 дней», дата сертификата, дневные лимиты) — Europe/Warsaw.
6. **Контент только через `lib/content`** с валидацией при старте. Битый контент → лог + понятная ошибка на странице, приложение не падает.
7. **Контракт контента** (docs/content-format.md) не менять в одностороннем порядке — согласовывать с course-factory.
8. **Конец каждой фазы:** обновлённый seed + smoke-чеклист, чтобы Иван проверил фазу за 10 минут.
9. **Определение «урок пройден»** (единственное, в т.ч. для сертификата): квиз сдан ≥2/3 И практика отмечена.
